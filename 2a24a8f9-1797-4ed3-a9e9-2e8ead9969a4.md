## 3. Supabase 및 개발자 우회 연동 디테일

* **로컬 환경 변수 설정 (`.env.local`)**:
  * `NEXT_PUBLIC_SUPABASE_URL` 및 `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 실제 동작하는 주소와 토큰으로 바인딩되어 활성화되었습니다.
  * 개발용 어드민 메일 설정: `NEXT_PUBLIC_DEV_ADMIN_EMAIL=benyny3325@gmail.com`
* **개발자 프리패스 가드 구현**:
  * 구글 로그인 시 DB에 회원 레코드가 생성되어 승인 완료 처리되기 전이라도, 로그인한 사용자의 이메일이 `benyny3325@gmail.com`인 경우 클라이언트(`AuthContext.tsx`)와 미들웨어 가드(`src/proxy.ts`)에서 무조건 **`admin` 역할** 및 **`is_approved: true`** 등급을 부여하여 로그인 대기실(`/pending-approval`)을 통과하고 어드민 승인 관리 등 모든 서비스에 무사 접근할 수 있도록 가드를 세팅했습니다.

## 4. Next.js 16 규격 변경 대응 (중요)

* **`middleware.ts` ➡️ `proxy.ts` 마이그레이션**:
  * Next.js 16 (React 19) 규격에서 기존 `middleware.ts` 파일 관례가 폐지(Deprecated)됨에 따라, 소스코드의 빌드 경고 해결 및 프로덕션 컴파일을 위해 기존 미들웨어를 삭제하고 최신 파일 구조인 **`src/proxy.ts`**로 온전히 리팩토링을 수행했습니다.
* **캐시 충돌 이슈 디버깅 가이드**:
  * 파일 리팩토링 직후 로컬 기동 시 Next.js가 `.next` 캐시 디렉토리에 이전 미들웨어 흔적을 남겨 두어 `Both middleware file and proxy file are detected` 에러를 뱉는 충돌 현상이 관측되었습니다.
  * **해결 방법**: 터미널에서 아래와 같이 **Next.js 개발 캐시 폴더를 완전히 삭제하고 개발 서버를 재실행**해 주어야 캐시 충돌이 해소됩니다.
    ```powershell
    # 프로젝트 루트(medical_project) 경로에서 실행
    Remove-Item -Recurse -Force .next
    npm run dev
    ```

1. **Google OAuth 인증 키 발급 및 Supabase Provider 연동**:

   * 구글 콘솔의 승인된 리디렉션 URI 목록에는 Supabase의 callback 주소인 `https://hygqahojxtycqeegifwr.supabase.co/auth/v1/callback` 를 반드시 화이트리스트 등록해야 합니다.
2. **Vercel 배포 및 도메인 바인딩**:
   * GitHub `main` 브랜치에 코드를 푸쉬하면 Vercel에서 실시간 자동 배포를 수행합니다.
   * Vercel의 **`Settings`** > **`Deployment Protection`** 메뉴로 진입하여 **`Vercel Authentication`** 토글 스위치를 `Disabled`로 꺼주셔야 접속 시 Vercel 가입 요구 화면 대신 정상 로그인 페이지가 열립니다.
   * 커스텀 도메인(예: 가비아 등 구매처) 연결 시 Vercel의 A 레코드(`76.76.21.21`) 및 CNAME(`cname.vercel-dns.com`)을 네임서버 설정에 등록하고, 최종 연결된 실서버 주소를 Supabase `URL Configuration` > `Site URL`과 `Redirect URLs`에 반영해주어야 정상 구글 로그인이 마무리가 됩니다.
